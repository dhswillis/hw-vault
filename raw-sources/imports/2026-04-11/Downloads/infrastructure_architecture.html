The router is a Python module that classifies each task and sends it to the cheapest model that can handle it. Runs as part of the API Wrapper on the VPS.

Cost impact: If 100% of tasks went to Claude Sonnet without caching: ~$50/mo in API calls. With this routing, ~75% of tasks go to Tier 1-2 (free or pennies), and Tier 3 uses prompt caching (90% input cost reduction). Estimated API cost: $10-15/mo.
How the router works: It's a Python class with a classify(task) method. Tasks come in with a type tag (from OpenClaw or the API Wrapper). The router maps task types to tiers, calls the appropriate model via OpenRouter (single API gateway for all models), and returns the response. Fallback: if a cheap model fails or returns low-confidence, auto-escalate to the next tier.
MODEL ROUTER — Task Classification Rules Tier 1 (FREE): task_type in ["parse_json", "format_output", "summarize_log", "extract_field", "template_fill", "data_transform"] Tier 2 (CHEAP): task_type in ["generate_code", "write_backtest", "analyze_results", "pattern_scan", "context_build", "report_generate"] Tier 3 (PREMIUM): task_type in ["strategy_evaluate", "setup_validate", "full_context_query", "knowledge_base_reason", "complex_analysis"] → Uses prompt caching: knowledge base as cached system prompt Tier 4 (CRITICAL): task_type in ["trade_signal_confirm", "risk_check", "live_execution_approve"] → Only fires when real money is on the line Auto-escalation: If response.confidence < 0.7 → escalate to next tier If response.error → retry once, then escalate If tier_4 fails → HALT, alert Harrison via Telegram
4. OpenRouter: One API for All Models
Instead of managing 5 different API keys and endpoints, OpenRouter provides a single gateway to every model.
Without OpenRouter
5 APIs to Manage
Anthropic API (Claude) — separate key, endpoint, SDK
xAI API (Grok) — separate key, endpoint, SDK
DeepSeek API — separate key, endpoint
Meta/Together (Llama) — separate key, endpoint
Mistral API — separate key, endpoint

Each with different auth, rate limits, error handling, retry logic. 5x the code, 5x the maintenance.
With OpenRouter
1 API, All Models
Single API key, single endpoint:
https://openrouter.ai/api/v1/chat/completions

Same request format for every model. Just change the model field:
• anthropic/claude-sonnet-4.5
• x-ai/grok-4.1-fast
• deepseek/deepseek-r1
• meta-llama/llama-3.3-70b

One error handler. One retry logic. One billing dashboard.
Exception: Claude prompt caching. For Tier 3/4 calls that use the 100K+ token knowledge base as a cached system prompt, call the Anthropic API directly (not through OpenRouter). Prompt caching saves 90% on input tokens and is only available via Anthropic's native API. The router handles this automatically — OpenRouter for Tiers 1-2, Anthropic direct for Tiers 3-4.
5. Backtrader: Free, Unlimited, Local Backtesting
Replace QuantConnect cloud dependency with a Python-native backtesting engine that runs on your Mac or VPS.
QuantConnect (Current)
Cloud-Dependent
✗ Runs on QC's servers — rate limited
✗ Researcher tier limits on compute
✗ Browser-based interaction (fragile)
✗ C# bias (Python support is secondary)
✗ Can't run locally for rapid iteration
✗ OOM issues with second-resolution data
✓ Good data library (futures, equities)
Backtrader (Target)
Local, Free, Unlimited
✓ Runs on Mac or VPS — no cloud needed
✓ No rate limits — run as many as you want
✓ Pure Python — matches your brain code
✓ Second-resolution data support native
✓ Parallel backtests on VPS (4+ cores)
✓ CSV, Pandas, IB data feeds
✓ Free, open source, well-documented
⚠ Need to source your own futures data
Data sourcing for Backtrader: You already have ES and NQ price data in ~/Downloads/tempo_extracted/price_data/ (1d and 1h). For second-resolution data, options include: (1) Databento — futures tick data, ~$50-100 for historical download, (2) Polygon.io — $99/mo for full futures, (3) Interactive Brokers — free with account, real-time + historical, or (4) export from Quantower's data manager (you already have Quantower). The cheapest path is exporting from Quantower since you're already paying for it.
BACKTRADER EXECUTION MODES Mode 1: Dev Iteration (Mac) Claude Code writes/edits brain → runs Backtrader locally → Quick 1-month test: ~30 seconds → Fast feedback loop for strategy tweaks Mode 2: Full Validation (VPS) OpenClaw triggers full backtest via Heartbeat → Multi-year run (2020-2026): ~5-15 minutes → Results analyzed by Tier 2 model (cheap) → Report sent via Telegram Mode 3: Parallel Strategy Sweep (VPS) Run 10-50 parameter variations simultaneously → 4 cores = 4 parallel backtests → Results aggregated, best params identified → Tier 2 model generates comparison report → Tier 3 (Claude Sonnet) validates winning config Mode 4: Walk-Forward Optimization (VPS, scheduled) Weekly: re-run backtest on latest data → Compare live results vs backtest predictions → Flag any divergence > 20% → Alert Harrison if strategy is degrading
Keep QuantConnect as backup. Don't delete your QC setup. Use it for: (1) comparing Backtrader results against QC's execution model, (2) accessing QC's built-in futures data library for validation, (3) any scenario where you want a second opinion on backtest accuracy. But primary iteration moves to Backtrader.
