# Plan: Build TempoIFVGv1.cs for NinjaTrader

## Overview
Build a NinjaTrader 8 strategy implementing the IFVG (Inverse FVG) entry from `TEMPO_IFVG_BUILD_SPEC.md`, using `SweepBreakv17.cs` as the structural template for unmanaged orders, stats display, and order lifecycle.

## Architecture
Single .cs file: `TempoIFVGv1.cs`, unmanaged orders, Calculate.OnEachTick, 1-minute chart.

## Implementation Steps

### 1. Scaffold from SweepBreakv17.cs
- Copy the class structure: internal types, state fields, lifecycle, OnBarUpdate, OnOrderUpdate, OnExecutionUpdate, stats display
- Rename to TempoIFVGv1
- Add all configurable parameters from the spec (GapMin, GapMax, MinRisk, MaxRisk, SweepMinPts, MaxBarsToIFVG, MaxBarsToFill, TP1Pts, TP2Pts, TP1Pct, TP2Pct, MaxConsecLosses, UseSoftStop, DisasterStopPts, level toggles)

### 2. Key Level System
- Build levels once per day (on new day detection)
- PDH/PDL: track yesterday's RTH (9:30-16:00 ET) high/low
- Asia H/L: 6:00 PM - 1:00 AM ET (previous evening)
- London H/L: 3:00 AM - 8:30 AM ET
- AM H/L: 9:30 - 10:30 AM ET (builds during the session)
- 1H Swing H/L: Use AddDataSeries(BarsPeriodType.Minute, 60) for 60-min bars, swing detection with lookback=3, last 5 days
- De-duplicate: merge levels within 2 points
- Each level tagged as "high" or "low" type

### 3. Sweep Detection
- On each bar close during entry window: check if bar swept above any "high" level and closed below → bearish sweep (SHORT setup)
- Or swept below any "low" level and closed above → bullish sweep (LONG setup)
- Min sweep wick = SweepMinPts (default 1.0)
- Store sweep info: direction, wick extreme, bar index, swept level

### 4. IFVG Detection (within MaxBarsToIFVG of sweep)
- After sweep detected, scan subsequent bars for 3-candle FVG pattern
- For SHORT setup (swept HIGH): look for bullish FVG (Low[0] > High[2]), then wait for close below FVG low
- For LONG setup (swept LOW): look for bearish FVG (Low[2] > High[0]), then wait for close above FVG high
- FVG gap size must be between GapMin and GapMax
- Store IFVG info: gap edges, bar index

### 5. Entry Confirmation (within MaxBarsToFill of IFVG)
- Wait for candle to CLOSE through the FVG
- LONG: close > FVG high → market order
- SHORT: close < FVG low → market order
- Stop = sweep wick extreme ± 1pt
- Risk filter: |stop - entry| must be between MinRisk and MaxRisk
- Check MaxConsecLosses hasn't been exceeded

### 6. Trade Management — Soft Stops + Partials
- **Soft stop**: on each bar close, check if close crossed stop level. If yes → exit at market
- **Disaster stop**: hard stop-market at stop + DisasterStopPts (safety net)
- **TP1**: when high/low reaches entry ± TP1Pts → close TP1Pct% of position, move stop to B/E (hard stop now)
- **TP2**: when high/low reaches entry ± TP2Pts → close TP2Pct% of position
- **Runner**: remaining position rides to FlattenTime
- Single contract mode: no partials, exit full at TP1

### 7. Position Sizing for Partials
- Base contracts = Quantity × 20
- TP1: close ceil(base × TP1Pct/100) contracts
- TP2: close ceil(base × TP2Pct/100) contracts
- Runner: remaining contracts
- Track partial state (0/2 partials taken)

### 8. Stats Display
- Same Consolas 11pt monospace box style as SweepBreakv17
- Sections: header, main stats, direction, today, open trade
- Add consecutive losses counter display
- Add partial status in open trade section
- Backtest/Live split when transitioning to realtime

### 9. Max Consecutive Losses
- Track per-day consecutive losses
- Win resets counter to 0
- B/E doesn't reset or increment
- When counter >= MaxConsecLosses, block new entries for rest of day
- Reset on new day
