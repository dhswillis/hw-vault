# Trading System Operations — Cluster Summary

> Operational backbone documents for the trading system: onboarding guides, session logs, repo inventories, research notes, and the two versions of Tempo rules extraction.

## Contents

This cluster covers the "how to use the system" layer — documents that tell a new Claude instance or Harrison himself where things are and how to get started. Includes:

**Onboarding:** TRADING_SYSTEM_START_HERE.md is the entry point for the trading system repo, covering file locations, architecture overview, and standing instructions. START_HERE.md (imports) is a duplicate. SETUP.md covers local dev environment.

**Session records:** SESSION_NOTES_FEB18.md captures a specific working session. CONGLOMERATE_SESSION_LOG.md tracks the Willis Holdings CEO agent session. RESEARCH_NOTES.md is a running journal of findings.

**Infrastructure:** repo_inventory.md catalogs the repo structure. BACKTESTING_AUDIT.md (imports) reviews the backtesting pipeline integrity.

**Tempo rules:** tempo_rules_v2_complete.md is the earlier extraction (V2); tempo_rules_v3_complete.md is the canonical 281-transcript version already summarized as [[wiki/summaries/tempo-rules-v3|Tempo Rules V3]].

## Status / caveats

Operational/contextual documents — low analytical value individually but important for system orientation. The START_HERE files contain duplicate content across three locations. The V2 rules are superseded by V3.
