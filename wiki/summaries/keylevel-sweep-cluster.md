# Key Level Sweep — NinjaTrader Build Cluster

> Build specs, working log, and V1 audit for the NinjaTrader implementation of the key level sweep fade strategy.

## Contents

Five documents covering the NinjaTrader implementation of the sweep fade research:

**NINJATRADER_BUILD_SPEC.md** (4,138 words) — Detailed build specification for the NinjaTrader strategy. Entry logic, key level computation, sweep detection, exit management.

**WORKING_LOG.md** (3,571 words) — Running development log tracking bugs found, fixes applied, and test results during the build.

**PORTFOLIO_AUDIT_V1.md** (2,612 words) — First audit of the implemented code against the spec. Bug inventory and verification checklist.

**COWORK_PROMPT.md + NINJATRADER_BUILD_PROMPT.md** — Prompts used to drive Claude Code/Cowork during the build. Operational scaffolding.

## Status / caveats

Implementation cluster. The research backing this is in [[wiki/summaries/sweep-fade-research|Sweep Fade Research]]. The overall sweep-fade strategy status is tracked in [[wiki/summaries/sweep-cluster|Sweep Cluster]] — most variants ended up dead or marginal.
